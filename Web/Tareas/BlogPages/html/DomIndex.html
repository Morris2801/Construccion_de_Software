Preguntas

  1. ¿Qué es el DOM?

    2. El DOM (Modelo de Objeto del Documento) es una representación en forma de árbol de un documento HTML. Permite que lenguajes de programación como JavaScript interactúen y manipulen los elementos de manera dinámica.

    Fuente
  2. ¿Qué es un nodo?

    3. Un nodo es cualquier componente del árbol del DOM. Puede representar diferentes partes del documento, como elementos HTML, atributos, texto y comentarios. Los principales tipos de nodos incluyen nodos de elementos (div, p, span, etc.), nodos de texto y nodos de atributos.

    Fuente
  3. ¿Cómo se accede a un elemento del DOM?

    4. Puedes acceder a un elemento del DOM utilizando métodos de JavaScript como:

    1. document.getElementById("id") → Selecciona un elemento por su ID.
    2. document.getElementsByClassName("class") → Selecciona elementos por su nombre de clase.
    3. document.getElementsByTagName("tag") → Selecciona elementos por su nombre de etiqueta.
    4. document.querySelector("selector") → Selecciona el primer elemento que coincide con un selector CSS.
    5. document.querySelectorAll("selector") → Selecciona todos los elementos que coinciden con un selector CSS.
    Fuente
  4. ¿Cómo se modifica un elemento del DOM?

    5. Puedes modificar un elemento del DOM cambiando sus propiedades, contenido o atributos utilizando JavaScript. Algunos ejemplos:

    let element = document.getElementById("myElement"); element.innerHTML = "Nuevo contenido"; // Cambia el contenido interno element.style.color = "red"; // Modifica el estilo element.setAttribute("class", "nuevaClase"); // Modifica un atributo Fuente
  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    6. Los métodos getElementBy* (como getElementById, getElementsByClassName, etc.) son más antiguos y están diseñados para seleccionar elementos específicos por ID, clase o etiqueta. Por otro lado, querySelector y querySelectorAll son más modernos y permiten seleccionar elementos utilizando cualquier selector CSS, ofreciendo mayor flexibilidad.

    Fuente

Diferencias entre getElementBy* y querySelector*

Criterio getElementBy* querySelector*
Compatibilidad Compatibles con navegadores más antiguos. Compatibles con navegadores modernos.
Selector Selecciona elementos por ID, clase o etiqueta específica. Utiliza cualquier selector CSS válido.
Retorno
  • getElementById: Retorna un único elemento.
  • getElementsByClassName y getElementsByTagName: Retornan una colección HTML.
  • querySelector: Retorna el primer elemento que coincide con el selector.
  • querySelectorAll: Retorna una NodeList con todos los elementos que coinciden.
Flexibilidad Menos flexible, ya que solo permite seleccionar por ID, clase o etiqueta. Más flexible, ya que permite usar combinaciones complejas de selectores CSS.
Uso común Útil para seleccionar elementos específicos de manera directa. Útil para seleccionar elementos con selectores complejos o múltiples condiciones.

Conclusiones

Experiencia general

Al utilizar ChatGPT y GitHub Copilot para contestar las preguntas y resolver los ejercicios, fue mucho más eficiente y rápida la resolución de las tareas. La capacidad de ChatGPT para generar explicaciones claras y ejemplos de código fue muy útil, especialmente al abordar conceptos complejos del DOM (los cuáles utilizabamos pero probablemente no comprendíamos). Por otro lado, GitHub Copilot ayudó completando los ejecicios de programación, estructurando hasta un .css ajustado al mismo .html. Poco a poco verificar lo realizado, tratar de entenderlo, y poder arreglar errores o inconsistencias, al igual que mezclando conocimiento propio con la visión personal de la solución es efectivo para aprender y practicar distintos conocimientos de manera dinámica.

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

Yo consisdero ChatGPT como una herramienta más creativa, particulamente pensando en texto. Ya sea traduciendo, parafraseando, o generando ideas, sus resultados me parecen más aptos y flexibles, mientras que con GitHub Copilot puedo aprovechar las opciones de autocompletar y vinculación de distintos archivos fuente para su interpretación de fuentes y lograr una interpretación de contexto más completa al momento de codificar. Asimismo, ya que tengo entendido que está diseñado a partir de bases de datos especializadas en programación, me parece que sus soluciones son más centradas en la tarea. La opción un poco más completa, desde mi perspectiva, es Copilot de Microsoft, ya que también funciona bastante bien para generación de imágenes (antes de probar el nuevo modelo de ChatGPT).

Ventajas de usar AI para programar

Para mí, utilizar Inteligencia Artificial para programar es más un recurso, así como al escribir a papel un diccionario sirve de apoyo. La AI puede ayudar a generar ideas, optimizar código, y resolver problemas de manera más rápida y eficiente, pero sin saber qué se está haciendo, si la herramienta comete un error, no se sabrá qué es o cómo solucionarlo. Por eso, prefiero a la par entender qué se está haciendo, y también primero resolver la tarea por mi cuenta antes de recurrir a tal modelo.

Desventajas de usar AI para programar

No obstante, el uso desmedido del recurso puede resultar contraproducente, ya que de vez en cuando llego a tener el impulso de querer solucionarlo todo con apoyo en una IA. De igual forma, a veces se llega a una solución sin entenderla por completo, contradiciendo el propósito de un "asistente" virtual. Más adelante, en un futuro académico y laboral, el estar al nivel de una IA, sin diferenciarme, no me traerá situaciones propicias a mis metas.
